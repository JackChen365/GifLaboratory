Examples, Tips, and References.

On this page I have collected some information which may be valuable, but which is not found in the documents distributed with Whirlgif 3.04.

Examples.

In my examples, I will give Unix style scripts. Personally, I prefer to use scripts. It makes it easier to debug the results.

Example 1.

On the internet a mail-to-me animated GIF has been floating around. It takes up 23,019 bytes, and shows a frog which eagerly eats letters as they are presented to it. I have used reanimations of it to hunt bugs in Whirlgif (my excuse is that most frogs in nature hunt bugs for a living).

The animation consists of 28 images. In the beginning, the frog just sits there and looks at a letter:

It is followed by a a series of images where the frog sticks out its tongue further and further. Finally it catches the letter:
Then the frog withdraws the letter and does what frogs usually do with what they catch, it eats it:

In the animation the gray color is chosen as a transparent background color. If I choose that each frame will be disposed by the background, each frame stands for it self. The background functions as an eraser.

The animation script looks as follows: 

#! /bin/sh
whirlgif -background 5 -loop 0 -trans 5 -disp back -o animfrog.gif \
-time 80 frog01.gif \
-time 20 frog02.gif frog03.gif frog04.gif frog05.gif frog06.gif frog07.gif \
   frog08.gif frog09.gif frog10.gif frog11.gif frog12.gif frog13.gif frog14.gif \
   frog15.gif frog16.gif frog17.gif frog18.gif frog19.gif frog20.gif frog21.gif \
   frog22.gif frog23.gif frog24.gif \
-time 15 frog25.gif frog26.gif frog27.gif \
-time 300 frog28.gif
exit

If you are interested, all the GIFs mentioned by name in the scripts on this page can be fetched with the HTTP-protocol - but I'm not telling you how ...

Example 2.

When I analyzed the frog-files, I found out that they actually used 12 different colors, but that the Color Map defined 256 colors.

In my favorite paint program, I reduced the Color Map - the paint program calls it the Palette - to 16 colors. I only bothered with editing the first frog and saved it as frog00.gif. Then I re-animated the whole thing using the -globmap option, which takes the Color Map from the first GIF and uses it in all the subsequent frames, i.e. Images.

The animation script now looks as follows: 

#! /bin/sh
whirlgif -globmap -background 5 -loop 0 -trans 5 -disp back -o froganim.gif \
-time 80 frog00.gif \
-time 20 frog02.gif frog03.gif frog04.gif frog05.gif frog06.gif frog07.gif \
   frog08.gif frog09.gif frog10.gif frog11.gif frog12.gif frog13.gif frog14.gif \
   frog15.gif frog16.gif frog17.gif frog18.gif frog19.gif frog20.gif frog21.gif \
   frog22.gif frog23.gif frog24.gif \
-time 15 frog25.gif frog26.gif frog27.gif \
-time 300 frog28.gif
exit

The new file is 19,236 bytes. The reduction in size - as well as time for loading - is some 20%. You will not see any difference between the two animations unless your browser treats 16-color maps different from 256-color maps.

The size is reduced by 3,783 bytes. This is more than the reduction of the global color map from 3*256 to 3*16 i.e. 720 bytes. - We have here one of many examples where a smaller color map saves more - 3,063 bytes - than the space saved by the color map alone. The explanation demands several words and some math. It is therefore left as a challenge for those interested in the details of how GIFs are packed.

Example 3.

To create an even smaller animation, I can use the -minimize option. I give up the transparent option, and repaint the gray background white in all the 28 frames. The disposal method will be set to the default of Whirlgif. If I show the animated frog on a white background and without a frame, few people will note the difference, even if it actually uses only 8 different colors. 

#! /bin/sh
whirlgif -globmap -minimize -loop 0 -o minifrog.gif \
-time 80 a00.gif \
-time 20 a01.gif a02.gif a03.gif a04.gif a05.gif a06.gif a07.gif \
   a08.gif a09.gif a10.gif a11.gif a12.gif a13.gif a14.gif a15.gif \
   a16.gif a17.gif a18.gif a19.gif a20.gif a21.gif a22.gif a23.gif \
-time 15 a24.gif a25.gif a26.gif \
-time 300 a27.gif
exit

The output from the examples 1-3 is shown on a separate page for comparison. The resulting size of the minimized frog is 4,892 bytes, compared to the original 23,019 bytes. I could have squeezed it a couple of bytes more, had I had the tools to make 8-color GIFs.

An other way to squeeze the animation is to consider the loop as a circle, where you can start anywhere. If I start with the simplest image of them all, a27.gif, and then continue with a00.gif et cetera, the animation gets smaller. The disadvantage is that the first image only shows the frog sitting there alone, and that image does not make sense as a mail-to-me image if presented by a browser which cannot show animations, or where animation is disabled.

Example 4.

The pipe smoker icon on my home page was made by copying the same image without smoke 30 times and then let the smoke develop from image to image. 

#! /bin/sh
whirlgif -minimize -time 25 -o ../images/pibe.gif -loop 0 /tmp/pibe???.gif 
exit

The resulting size is 2,685 bytes. Without the -minimize option, it would be 8,715 bytes.

Example 6.

I have been asked whether minimizing and a transparent background can go together. They can! An example is this picture of a simple "cartoon" of the sort that many of us created during boring school hours. I made it up as a logo for Whirlgif rev 3 (like Kevin Kadow's reversly turning earth is a logo for Whirlgif rev 2). There are 12 frames, and the script is: 

#! /bin/sh
whirlgif -loop 0 -minimize -trans 4 -background 4 -o bladbog1.gif \
      -time 10 bbog??.gif
exit

The trick is to have some non transparent raster data replace the previous data. The little pencil is not only there to show the tools which the "artist" used. Some of it replaces the upper corner of the falling page.

Tips.

Whirlgif in all releases talks GIF! The input files are GIFs and the output file is a GIF. The program may be called crude in the sense that if the input files are garbage, the output will be garbage.

The Structure of An Animated GIF.

If you have a fair idea of the structure of a GIF-file, skip this section, as it will not go into details.

Two GIF formats exist, called GIF87a and GIF89a.

Overview of the GIF87a Format.
Mandatory
GIF SignatureContains the text GIF87A
Mandatory
Screen DescriptorThe size of the GIF screen i.e. the total size to be set off to all images below. Whether or not a Global Color Map follows - and if so - the number of colors used in the Global Color Map and the color used as background color.
Optional
Global Color MapThe default colors to be used in the images.
Repeated as many times as you need
The Local Color Map is optional
 Image DescriptorThe size and the position of the following image. Whether or not a local color map should be used, and the size of it.
Local Color MapThe colors to be used in the following image.
Raster DataEach pixel of the image is described by its color. Data are compressed (LZW).
Mandatory
GIF TerminatorEnds the GIF file

The Color Maps are optional. The GIF87a specifications state that a Default Color Map should be used, if all Color Maps are absent. It says nothing, however, about what a Default Color Map is. Whirlgif needs at least one of them.

As you may understand, it is possible in a GIF87a file to define some sort of a bulletin board, with several pictures (images) on it. Very few GIF viewers can handle more than one image, and perhaps only two of them will show the total screen with its background color. Browsers like e.g. Netscape, Opera, and the Internet Explorer can to some extend. I had a little problem with example 5 above and have made a note ³) about the treatment of a large background with a small image.

The above does not give all variants of the GIF89a formats. If you want to know more, follow the links in the second column. The terms used throughout these notes and in the program texts differ from those of the GIF89a specifications, as earlier programmers and I have chosen to stick to the GIF87a term for equivalent portions of GIFs.

Colors.

Each Color Map takes up 3 times as many bytes as the number of colors. Color Maps in GIF files can have 256, 128, 64, 32, 16, 8, 4, or 2 colors. Most paint programs can only write GIFs indicated by the boldface numbers. Whirlgif can read and write them all but depends on the abilities of your paint program(s), which often call the Color Map your palette.

If your aim is to make relatively small GIFs, it is a good rule to use as few colors as possible. In animated GIFs use the same Color Map throughout your images.

The -globmap option minimizes the amount of Color Maps by remapping each pixel to the nearest color in the Global Color Map. In this program it is a pure geometrical approximation. Often it is a good approximation. View your animation carefully, however, if you use the -globmap option. The beauty is in the eye of the beholder.

If you are unhappy with the result, try first to edit the colors of the first image. Ultimately, you may have to reduce the number of colors in each frame by means of your favorite paint program. For certain GIFs this may not drastically reduce the size of the file. The reason is that the program may dither the colors to look almost unchanged. The LZW compression algorithm gives the best results with relatively large areas of either the same color or with a systematic pattern. Dithering, however, usually consists in making an irregular pattern of color dots which to the eye blends to something acceptable.

The Smallest Difference.

The -minimize option makes Whirlgif find the smallest difference between two successive images, and then deliver the smallest rectangle covering the differences together with the position of the rectangle. It can give some really small animations.

If you try to optimize the size of an animated GIF consisting of dithered images, the reduction in size may not be very astounding. A random dithering will usually yield a very large area with different random patterns.

Things to be Careful With.

My advice is to set the background color and the transparent color to the same value. Neither Netscape nor the Internet Explorer follow the specifications as I read them, and they differ somewhat. The Norwegian browser Opera tries it best to be bug to bug compatible with Netscape.

If you want to create a bulletin board by displaying one or more images within the same (large) Screen, preferably use Color Map index [0] as your background, and use the trick outlined in Example 5 above.

Avoid the disposal method -disp prev. The result looks different in Netscape and the Internet Explorer.

If you intend to present your animation on a web page you should always give the size of the image.

e.g.    <IMG SRC="animfrog.gif" HEIGHT=33 WIDTH=585>
Several versions of the he Internet Explorer have problems figuring out the size. They need your help.

References to Other Information

 Most of he information is collected while I worked with Whirlgif 3.00. I lost my earlier bookmarks, and I badly miss a reference to an Italian(?) site which contains comments about how GIF animations can be made - also with Whirlgif. Please mail to me or write it in the list of references, tips etc, if you know something about it.
The newsgroup alt.binaries.pictures.animated.gifs brings a steadily flowing stream of animated GIFs.
The group attracts a lot of friendly people. Many of them will help you, if you need a banner, an animation of a love letter, or whatever. Lurk a while, get ideas, and enjoy their friendlines.
The official FAQ pages of the newsgroup come with references to various sources for animated GIFs.
Of course it contains lots of multi media and animated illustrations. It is an impressive experience which can give you many ideas. The FAQ is also published as a plain text and in a pure HTML version which is the form I prefer. Perhaps because I have to pay for the load time of the big one over my telephone bill.
Gifsicle, by Eddie Kohler et al. is a true competitor.  Like Whirlgif it is command line driven
and the input and output files are GIFs.  Gifsicle is freeware and uses the same fast & free LZW encoder 4) as Whirlgif.  Gifsicle has a lot of options - many more than Whirlgif.  Gifsicle's --optimize function may create smaller GIFs than Whirlgif's -minimize option.  Gifsicle can perform many of the functions where Whirlgif needs a paint program.
Barry's Clip Art Server is an example of a source where animations are offered to the public.
The pages are beautifully ordered, and it should be easy to find your way to the type of animation that you need. Barry generates his pages on a MAC.
IrfanView by Skiljan Irfan is my favorite viewer for a quick view at an animated GIF ...
even if an animation sometimes is displayed somewhat flickering when the animation uses the disposal method "restore to background". The flickering is caused by Irfan wanting to show the background a short while before the next image, IrfanView is Freeware for non commercial use. Irfan knows a lot about image formats, and the program is maintained on a regular basis. The program is for Windows95/98/NT only.
